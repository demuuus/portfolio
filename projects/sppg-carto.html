Badan Gizi Nasional - SPPG Carto

Overview

SPPG Carto is a geospatial web application for visualizing and managing SPPG (Satuan Pendidikan Penyelenggara Gizi) regional planning data on an interactive map. It allows regional administrators to view, filter, and analyze nutrition program coverage across districts.

Problem

Regional nutrition program planners were working with spreadsheets and static PDFs to track coverage data across dozens of sub-districts. There was no easy way to visualize which areas were underserved or to spot geographic patterns in the data.

Solution

A React-based web app with an interactive Leaflet.js map that plots SPPG data points geographically. Users can filter by district, view coverage statistics in a side panel, and export filtered data. GeoJSON boundaries are overlaid for clear regional context.

Key Features

  • Interactive map with zoomable GeoJSON district boundaries
  • Color-coded markers by coverage status and category
  • Side panel with filtering, search, and summary statistics
  • CSV data import and export
  • Responsive layout for tablet use in the field
  • Offline-capable with locally cached map tiles

Technical Challenges

Rendering hundreds of GeoJSON polygons smoothly on a Leaflet map was a performance challenge. I solved this using layer grouping and only rendering visible viewport layers, which significantly improved scroll and zoom performance.

The GeoJSON data for Indonesian sub-districts was also inconsistent — some boundaries had gaps, overlaps, or coordinate precision issues that needed preprocessing before they could be rendered cleanly.

Impact

The app was deployed for use by regional planners and replaced a manual spreadsheet workflow. It reduced the time to identify underserved areas from hours of spreadsheet analysis to a few seconds of map interaction.
