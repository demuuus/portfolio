Adhan Caller

Overview

Adhan Automation Server is a lightweight Python-based system that automatically plays the Islamic call to prayer (adhan) at the five daily prayer times. It fetches accurate timings from the Aladhan API based on a configured city and calculation method, schedules playback, and runs continuously as a background service on a Linux machine.

The system is designed to run on low-power devices such as old laptops or home servers, providing a reliable and low-cost adhan solution without requiring specialized hardware.

Motivation

This project was built as a practical automation system to solve a real-world need: having a consistent, automated adhan player at home without relying on mobile apps or expensive dedicated devices.

It also served as a hands-on way to explore:

  • Linux system services (systemd)
  • Automation scripting in Python
  • Real-time scheduling and reliability under unstable network conditions

How It Works

The system runs as a background service on a Linux machine. When it starts, it fetches prayer times from the Aladhan API using the configured location and stores them locally in a cache file. This allows the system to continue functioning even when offline. A scheduler continuously runs in the background, triggering a reminder audio two minutes before each prayer and the full adhan exactly at the scheduled time. The system automatically refreshes prayer times daily and includes a retry mechanism to handle temporary network failures, ensuring that schedules are eventually updated without manual intervention.

Key Features

  • Scheduling & Automation which plays automatic adhan playback for all five daily prayers, pre-adhan reminder (2 minutes before each prayer), and daily schedule refresh with retry mechanism
  • Network Resilience which automatically retry when internet connection is restored, Offline mode using cached prayer times (cache.json), and No manual restart required after network failure
  • Smart Refresh Logic which prevents unnecessary API calls (refreshes only once per day), detects outdated cache and updates automatically, and handles missed midnight refresh due to downtime
  • Audio System uses mpg123 for reliable MP3 playback and supports custom adhan and reminder audio files
  • System Integration runs as a systemd service, auto-starts on boot, and automatically restarts on failure
  • Monitoring & Debugging

Hardware

This system is designed to run on:

  • Old laptops repurposed as home servers
  • Linux machines (Ubuntu/Debian-based)
  • Any device capable of running Python + systemd

What I Learned

Through this project, I gained practical experience in building a long-running automation system that must remain stable over time. This included working with systemd services, handling scheduling logic, and designing systems that can recover from failures without user input. A significant part of the learning process involved dealing with real-world edge cases such as missed scheduled events, stale cached data, and unreliable network connectivity.

One of the most challenging aspects was ensuring that the system remains accurate and self-correcting. Implementing a retry-based refresh mechanism and tying refresh logic to persistent state (rather than in-memory variables) were key steps in making the system robust. Debugging through logs and system-level tools like journalctl also played an important role in understanding how the system behaves in production.
