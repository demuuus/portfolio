Adhan Caller

Overview

Adhan Caller is a Raspberry Pi-powered device that automatically plays the Islamic call to prayer (adhan) at the five daily prayer times. It calculates accurate prayer times based on GPS location and the selected calculation method, then triggers audio playback through a connected speaker system.

Motivation

I built this as a practical project to learn embedded systems and also as a personal gift for my grandparents, who wanted an automated adhan caller for their home but couldn't find an affordable and customizable solution locally.

How It Works

The system runs a Python script on boot that fetches prayer times from the Aladhan API for the configured location. Times are stored locally in SQLite so it works without internet after initial sync. A scheduler triggers aplay at each prayer time to play the selected adhan audio file through the audio output.

Key Features

  • Automatic prayer time calculation based on GPS coordinates
  • Multiple calculation methods supported (MUI, ISNA, Egyptian, etc.)
  • Offline mode — times cached locally in SQLite
  • Configurable adhan audio files per prayer
  • Volume control and silent mode schedule
  • Web UI for easy configuration (no SSH required)
  • Auto-starts on boot via systemd service

Hardware

  • Raspberry Pi 3B+
  • USB audio adapter
  • Passive speaker with amplifier board
  • 3D-printed enclosure

What I Learned

This project was my first hands-on experience with embedded Linux and hardware interfacing. Managing audio on Linux, handling systemd services, and building a lightweight web UI for config were all new territory for me. The hardest part was getting reliable audio playback without glitches at exact scheduled times.
