Tuschology-IDX-Bot

Overview

Tuschology-IDX-Bot is a real-time market intelligence bot that monitors Indonesian stock market activity by combining official IDX (Indonesia Stock Exchange) announcements with curated financial news feeds. It delivers structured alerts to both Discord and Telegram, enabling retail investors to stay informed without manually tracking multiple platforms.

Problem

Indonesian retail investors often miss critical market-moving events such as suspensions, unusual market activity (UMA), and corporate disclosures because information is fragmented across the IDX website and various financial news portals. Monitoring these sources manually is inefficient and increases the risk of delayed decision-making.

Solution

This bot continuously runs asynchronous pipelines that:

  • Scrape official IDX announcement pages for high-priority market events
  • Aggregate financial news from multiple RSS feeds
  • Filter and deduplicate content in real-time
  • Deliver structured alerts instantly to Discord and Telegram

The system also adapts its behavior based on market conditions, including maintenance detection and trading hours awareness.

Key Features

  • IDX Priority Monitoring which detects and categorizes Suspensions, Unusual Market Activity (UMA), Disclosure, General issuer announcements
  • Multi-source RSS Aggregation which integrates with CNBC Indonesia, CNN Indonesia, Tempo, Antara, Kontan, and Bloomberg Indonesia
  • Real-time Async Processing
  • Deduplication System which uses hashed URLs and in-memory cache to prevent duplicate alerts
  • Discord Integration with topic based routing, multi-server support, dynamic rich presence based on market state
  • Telegram Integration with broadcast alerts, status command, real-time system metrics
  • State Persistence which is JSON-based storage for sent IDX announcements and last seen RSS entries

Technical Challenges

One of the main challenges was handling dynamic and inconsistent IDX website structures, where traditional scraping selectors often failed due to frontend changes. This was solved by implementing flexible anchor-based parsing combined with path filtering.

Another challenge was ensuring system stability in an always-on asynchronous environment. Without cron-based execution, failures could silently stop pipelines. This required careful exception handling, state persistence, and process management strategies.

Additionally, managing real-time deduplication across multiple heterogeneous data sources required a reliable hashing strategy to avoid repeated alerts while maintaining performance.

What I Learned

This project deepened my understanding of:

  • Asynchronous programming in Python for real-time data processing
  • Real-time data pipelines vs scheduled jobs
  • Multi-platform bot architecture (Discord + Telegram)
  • Resilient web scraping under changing frontend structures
  • State management without traditional databases

It also introduced practical challenges in deploying and maintaining long-running services, including process supervision, error recovery, and production stability.
