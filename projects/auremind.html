College Project 2025

AureMind

Overview

A desktop application developed as part of a database systems and software engineering course. The project demonstrates practical implementation of relational database concepts, CRUD operations, and a JavaFX graphical user interface.

Project Brief

The assignment required building a complete data management system from scratch, including database design, normalization, query optimization, and a user-facing GUI. The system had to support multiple user roles with different access levels.

What Was Built

A student records management system with separate admin and user interfaces. Admins can manage student data, course enrollments, and generate reports. Regular users can view their own records and submit requests.

Key Features

  • Relational database with normalized schema (3NF)
  • Role-based access control (admin vs. student)
  • JavaFX UI with table views, forms, and dialogs
  • CRUD operations for students, courses, and enrollments
  • Report generation with filtered exports
  • Password hashing for user authentication

Database Design

The schema was designed to 3rd Normal Form to eliminate redundancy. Key tables include Students, Courses, Enrollments, Users, and AuditLog. Foreign key constraints and transactions ensure data integrity across operations.

Outcome

Received full marks for the project. The main feedback from the lecturer was positive on the normalization and transaction handling, with a suggestion to add indexing on the frequently queried columns — which I implemented as an extra improvement.
